# Tools
<p>
  Minhas "tools" são ferramentas que facilitam e agilizam (ou já facilitaram e
  agilizaram algum dia) um problema do dia a dia.
<p>
<p>
  Dessa forma, consigo ao mesmo tempo que, resolver um problema específico meu,
  aprender lógica de programação de uma forma inovadora!
<p>

## colors.py
<p>
  Essa ferramenta foi criada durante meu tempo como membro da FranRobots,
  ela serviu como uma forma de criar "vítimas" (alvos do robo) aleatóriamente,
  de uma forma mais rápida e prática, já dizendo diretamente o valor dela, as cores em ordem.
<p>
<p> 
  Isso ajudou a minha equipe, a fazer a montagem das vítimas de uma forma muito mais veloz,
  sem ter que pensar ou pesquisar cada vítima e calcular se ela é verdadeira ou falsa.
<p>

## renomear_fotos.py
<p>
  Esta também é uma ferramenta que foi desenvolvida durante meu período como membro da equipe de robótica.
  Assim como o nome diz, ela serve para renomear os arquivos dentro de uma pasta, para letras aleatórias,
  isso ajudou muito na organização do banco de dados da Machine Learning do robô, isso pois ao tirar fotos
  para o reconhecimento de imagem, o robô tirava fotos de 30 em 30, salvando como "foto_1, foto_2, ...",
  isso fazia com que eu tivesse que criar pastas a cada 30 fotos, para nao ter conflito de arquivos com o mesmo nome.
<p>
<p>
  A solução foi justamente esta ferramenta, com ela, bastava colocar o caminho e as fotos eram todas renomeadas,
  podendo assim, ter maior organização na separação de fotos para o banco de dados.
<p>

## RandomSum.py
<p>
  Esta ferramenta serve para gerar contas de soma com números aleatórios.
  Apesar de simples, ela me facilitou a gerar contas simples de 1 a 5 digitos, que me ajudaram a estudar somas pequenas de cabeça, com digitos aleatórios.
<p>

## insertQueryGenerator(V3).py
<p>
  Assim como o nome já diz, é uma ferramenta que gera querys de INSERT para um banco de dados (MySQL).
  Essa ferramenta, serve para quando você precisa de uma query de insert, que insira muitos valores de uma só vez
  por motivos de testes locais (por exemplo criar 100 usuários).
  Ou seja, essa ferramenta facilita e agiliza a criação de uma query muito longa, sem ter que digitá-la a mão.
<p>
<p>
  O diferencial, é que na V3, há a possibilidade de escolha entre colocar um prefixo em cada um dos atributos e 
  escolher o tamanho do contador (sufixo), dessa forma, você consegue manter um tamanho padrão (bom para cpf que tem sempre 11 digitos),
  além de colocar prefixos para facilitar nos testes (ex: "nome1001, nome1002").
<p>
